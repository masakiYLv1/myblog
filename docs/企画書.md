# 企画書 (生成 AI と共に作成)

## **サービス名: myBlog**

---

## **1. サービス概要（Summary）**

このサービスは、技術記事を投稿・共有できる **Zenn 風の記事プラットフォーム**です。

Markdown で記事を書き、タグで分類し、閲覧者は記事の一覧・詳細を見られます。

ユーザー登録すると自分のページ（プロフィール / 投稿一覧）を持つことができます。

---

## **2. 背景（Why / Problem）**

### ■ なぜ作るのか？

- 技術記事をストレスなく書ける UI・機能を再現したい
- Markdown 記法を使った記事投稿システムを学びたい
- 認証・CRUD・多対多など、実務レベルの機能を一通り習得したい

### ■ 解決したい課題

- 個人ブログは面倒、Qiita は機能が重い → “ミニマムな記事プラットフォーム”を作りたい
- markdown → 表示までの仕組みを学ぶ機会が少ない
- Web アプリの設計〜実装まで通しで経験したい

---

## **3. 想定ユーザー（Target）**

- エンジニア / エンジニア志望の学習者
- 技術メモを簡単にまとめたい人
- ポートフォリオ目的で記事を公開したい人

---

## **4. サービスの目的（Goal）**

- Markdown で技術記事を書き、公開できる
- タグによる分類で記事を探しやすくする
- ユーザーが自分の記事一覧ページを持てる
- シンプル・高速・使いやすい記事投稿体験を提供する

---

## **5. 主要機能（MVP Scope）**

### 🔹 **閲覧系（誰でも閲覧可能）**

- 記事一覧表示
- 記事詳細（Markdown 表示）
- タグ一覧
- タグ別記事一覧
- ユーザーのプロフィール + 投稿一覧

### 🔹 **投稿系（ログインユーザー）**

- ログイン / ログアウト（Supabase Auth）
- 記事投稿（Markdown 形式）
- タグの選択 / 作成
- 下書き保存（非公開）
- 記事編集 / 削除

### 🔹 **管理系（本人記事のみ編集可能）**

- RLS（Row Level Security）でアクセス制御

---

## **6. 非機能要件（NFR）**

### ■ パフォーマンス

- Vite による高速開発環境
- キャッシュにより記事表示を高速化

### ■ セキュリティ

- Supabase Auth による認証
- RLS による投稿者以外編集不可

### ■ デプロイ

- Firebase Hosting
- GitHub Actions による CI/CD 自動化

### ■ テスト

- Jest + React Testing Library
- 投稿フォームやページ遷移

---
